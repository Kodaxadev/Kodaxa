# Content Truth Register

This document controls how the Kodaxa portfolio describes each repository. The website should communicate demonstrated work without implying a deployment or maturity level that has not been shown by the corresponding project repository.

| Project | Portfolio classification | Public wording currently allowed |
| --- | --- | --- |
| FrontierWarden | Featured system / testnet infrastructure | Proof-backed trust and credit-risk infrastructure; live on Stillness/testnet as documented by its repository; binding-state limitations remain visible in detailed coverage. |
| Signal Vault | Alpha product | Local-first field intelligence product; browser, supported object route, desktop companion, and optional API surface; do not claim public remote-write readiness until deployment verification exists. |
| EF-Atlas | Deployed product | Authority-tiered EVE Frontier builder knowledge corpus and API; link to `atlas.kodaxa.dev`. |
| Code-Warden | Published technical toolkit | Governance and verification layer for AI coding agents; do not overstate unsupported runtime enforcement. |
| STEP | Live application | Procedure-first GED Math learning application; link to `step.kodaxa.dev`; preserve local-only/no-account positioning. |
| RowDiFF | Technical tool | Semantic CSV comparison and merge tooling with Git integration. |
| ChronosMCP | Technical tool | Local MCP memory and structured project-context layer. |
| Task Anchor | Technical tool | MCP task-lock and drift-handling system for AI-assisted workflows. |
| Agency Terminal | Concept study | Documented tribal operations/evidence-ledger concept suite; never label as a deployed application without new evidence. |

## Copy principles

1. Prefer boundaries over hype.
2. Show status alongside project identity.
3. Link live products only when a known public URL exists.
4. Use system diagrams to explain what was built, not to imply integrations that are not live.
5. Update this register before changing claims in case-study content.
